# Kanban Travel Schedule Enhancement Prompt

## 目標
為 Docusaurus kanban 格式的旅行計劃檔案加入天氣資訊和日出日落時間

## 步驟

### 1. 分析現有檔案
- 檢查 kanban-plugin: board 格式的 markdown 檔案
- 識別每日行程的結構和地點

### 2. 天氣資訊格式
```markdown
- [ ] 🌡️ **地點** 最高溫°C/最低溫°C 表情符號 | **地點2** 最高溫°C/最低溫°C 表情符號
```

### 3. 日出日落時間格式
```markdown
- [ ] 🌅🌇 **日出日落** 地點 時間/時間 | 地點2 時間/時間
```

### 4. 溫度表情符號分級
- 🔥 炎熱 (35°C+)
- 🌡️ 很熱 (30-34°C)
- ☀️ 熱 (25-29°C)
- 🌞 溫暖 (15-24°C)
- 🌤️ 涼爽 (5-14°C)
- 🌥️ 冷 (-1-4°C)
- ❄️ 寒冷 (-2°C 到 -14°C)
- 🥶 極寒 (-15°C 到 -22°C)
- 🧊 酷寒 (-24°C 以下)

### 5. 卡片順序統一
每日格式：
1. 圖片卡片
2. 溫度資訊卡片
3. 日出日落時間卡片
4. 其他行程卡片

### 6. 參考區段增強
加入旅行建議和溫度警告：
```markdown
- [ ] **重要溫度提醒**：
	- 最極端：地點 溫度 表情符號
	- 溫差最大：從地點1 溫度 到地點2 溫度，溫差 XX°C
	- 日照時間：約 X-X 小時
- [ ] **旅行建議**：
	- 準備分層保暖系統，應對巨大溫差
	- 特別注意高海拔地區的極低溫
	- 利用較晚的日出時間安排行程
	- 日落後溫度急降，需提前保暖
```

## 使用方式
1. 提供旅行計劃的 markdown 檔案
2. 說明：「請仿照 kanban-prompt.md 的格式，為這個旅行計劃加入天氣資訊和日出日落時間」
3. AI 會自動分析地點、查找天氣資料、統一格式並加入表情符號

## 注意事項
- 保持 kanban-plugin: board 格式不變
- 溫度資料需根據旅行月份調整
- 日出日落時間考慮地理位置和季節
- 表情符號要與溫度範圍對應
- 統一所有日期的卡片格式
