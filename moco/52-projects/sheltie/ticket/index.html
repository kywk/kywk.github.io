Ticket

播放模式下:

  • 字體再放大些
  • 時程長度依實際需要日期比例
  • 幫我挑七個時程顏色, 各階段依序使用, 第八個顏色與第一個顏色相同

專案 Markdown 的時程, 不要局限 開發/SIT/QAS/REG/PROD, 可以自定義, 並顯示在投影片上.

編輯模式下, 點擊預覽的投影片, 左邊編輯區的游標可快速跳到對應的 markdown 區塊.

  • 投影片播放, 放大中央投影片內文字和必要圖示大小, 減少白底, 避免離螢幕比較遠的人員看不清楚
  • 投影片播放, 放大中央投影片比例, 減少外框占比
  • 投影片 Layout 調整如下
    • 狀態, 進度, 窗口, 分類 用類似 tag 的方式標示在專案名稱之後
    • 移除主辦 和時程之間的分隔線
  • 協辦可能有相當多單位, 請確保畫面配置得當
  • 請檢視並調整投影片的版面設計, 清楚呈現資訊同時保持版面整潔

無痕模式初次載入有些緩慢, 請協助釐清效能問題並修復

請提供線上投影片播放的功能

會辦狀況請比照下列格式處理, 保留純 Markdown 時仍可清楚條列

- YYYY-MM-DD
  - 會辦狀況1
  - _待追蹤_ 待追蹤事項
  - _預計_ 下週開始開發

Ex:

- 2025-12-20
	- _待追蹤_ 需要確認規格文件
	- 完成初步設計評審
- 2025-12-15
	- _預計_ 下週開始開發
	- 正在進行需求訪談
- 2025-10-01
	- 這是一個月前的舊記錄
	- 應該顯示為灰色

協同工作調整

  • 連線中燈號直接顯示在 "🐕 工作區名稱" 之後
  • 多人連線時, 仿照 GoogleDoc 燈號自動挑選個顏色/圖示表示不同人
  • 多人連線時, 在編輯區顯示不同人的游標位置

專案 Markdown 範例資料請參考 @sample.md, 說明如下:

  • 時程固定為 開發 > SIT > QAS > REG > PROD
    • 生成的投影片可參考 @border-collie 的箭頭甘特圖, 並加上 today pin
    • 若太複雜請設計適當的 UI/UX 顯示方式
  • 專案投影片基本資訊欄位需要放相關單位, 長官需知道
  • 文字/WYSIWYG 的切換 UI 請比照 @border-collie 的 文字/表單
  • 會辦狀況內容的處理有誤
    • 內容沒有固定格式, 無須進行後處理, 保持純文字內容同步與顯示
    • 待追蹤 開頭的項目, 文字為紅色/粗體
    • 預計 開頭的項目, 文字為藍色/斜體
    • 一個月前的會辦資訊字體顏色為灰色
    • 會辦內容沒有即時更新預覽

  • 工作區 / 預覽區的工具列 UI 配置比照 @border-collie
  • 提供編輯區和檢視區 toggle 按鈕, 可完全收縮放到左邊(編輯區) 和右邊(檢視區), 留下 Toggle 按鈕位置
