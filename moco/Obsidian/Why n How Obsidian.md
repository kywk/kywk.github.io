# [Obsidian] Why & How - 選擇與使用指南

在知識管理工具的選擇上，經過充分的比較和考量，最終選擇了 Obsidian 作為主要的筆記和知識管理平台。

## 為什麼選擇 Obsidian

### 📼 需求背景
工作上有多個專案與代辦事項需要統整，多個臨時念頭需要紀錄，決定找個筆記軟體好好深入使用。恰好遇上 [Obsidian 1.0](../../../news/2022/10-13-obsidian-1-0) 發佈，而無所不知的 Google 也在這段時間塞了一堆 Logseq 的介紹。

### 🔍 筆記軟體比較

#### Notion vs Obsidian
**Notion 的優勢**：
- 功能強大，支援資料庫、看板、協作等
- 線上協作方便
- 美觀的介面設計

**為什麼沒選 Notion**：
- **本地儲存需求**：作為長期個人知識庫，能使用本地端儲存是相當重要的
- **穩定性疑慮**：線上服務的不穩定性還是偏高
- **數據所有權**：無法完全控制自己的數據

#### Logseq vs Obsidian
**Logseq 的優勢**：
- 開源軟體，社群驅動
- 支援線上版本
- Block-based 的編輯方式

**為什麼選擇 Obsidian**：
1. **速度優勢**：Obsidian 在效能上明顯優於 Logseq
2. **生態系統**：更成熟的插件生態系統
3. **定製化**：透過「折騰」可以彌補 Obsidian 罩門，使其變成更好的選擇

### 🎯 決定因素
- **本地儲存**：所有數據都在本地，完全控制
- **Markdown 格式**：開放標準，不被特定平台綁定
- **擴展性**：豐富的插件生態系統
- **效能**：快速的啟動和運行速度
- **學習曲線**：適中的學習難度，不會過於複雜

## 如何使用 Obsidian

### 📝 筆記編輯器
**統一的 Markdown 編輯環境**
- 所有文檔都使用 Markdown 格式
- 支援實時預覽和源碼模式
- 豐富的編輯功能和快速鍵

### 📊 通用筆記系統
**隨時記錄，隨時整理**
- **快速捕捉**：使用 [[Obsidian Daily Notes]] 記錄日常想法
- **主題整理**：將相關內容整理成主題文章
- **連結建立**：使用 wikilink 建立知識間的連結
- **標籤分類**：使用標籤系統進行分類管理

### 📊 習慣追蹤
**量化自我管理**
- 使用 [[Obsidian Plugin Tracker]] 追蹤健康數據
- 在 [[Obsidian Daily Notes]] 中記錄每日指標
- 透過 [[Obsidian Plugin Dataview]] 分析趨勢
- 建立個人化的追蹤系統

### 📋 任務管理
**GTD 方法論實踐**
- 使用 [[Obsidian Plugin Tasks]] 管理任務
- 透過 [[Obsidian Plugin Kanban]] 視覺化工作流程
- 在日記中整合任務查詢
- 建立個人任務管理系統

### 📁 專案管理
**知識工作者的專案管理**
- 為每個專案建立獨立資料夾
- 使用 [[Obsidian Plugin Templater]] 建立標準化模板
- 透過標籤和連結建立專案關聯
- 利用 [[Obsidian Plugin Dataview]] 生成專案報告

## 我的 Obsidian 環境

### 🔧 核心配置
- **主題**: Blue Topaz 主題
- **快速鍵**: 自訂左右側邊欄切換 (`⌥ ←` / `⌥ →`)
- **資料夾結構**: 按功能和專案分類組織

### 🔌 必備插件
- **[[Obsidian Plugin Templater]]**: 動態模板系統
- **[[Obsidian Plugin Dataview]]**: 數據查詢和分析
- **[[Obsidian Plugin Tasks]]**: 進階任務管理
- **[[Obsidian Plugin Kanban]]**: 視覺化工作流程
- **[[Obsidian Plugin Tracker]]**: 數據追蹤和視覺化

### 📅 工作流程
1. **每日開始**: 開啟當日日記，檢查任務和行程
2. **隨時記錄**: 在日記中記錄想法、會議、學習心得
3. **定期整理**: 每週整理重要內容，建立主題文章
4. **數據分析**: 每月分析習慣和目標達成情況
